## <a id="описание"></a> 📋 Описание
Этот проект преобразует "сырые" CSV-данные о ценах на недвижимость в красивый, интерактивный дашборд Tableau. Он позволяет исследовать ценовые тренды на первичном и вторичном рынках жилья в разных регионах, фильтровать по классу недвижимости и выявлять рыночные закономерности быстрее, чем риелтор успеет сказать "уникальное предложение"! 🤖

## <a id="возможности"></a> ✨ Возможности
- **Интерактивный дашборд** — Фильтруйте данные по региону, дате и типу недвижимости одним кликом
- **Динамика цен** — Отслеживайте изменение цен за квадратный метр во времени
- **Сегментация рынка** — Отдельные представления для первичного и вторичного рынков
- **Сравнение регионов** — Сравнивайте ценовые тренды в более чем 80 регионах России
- **Чистый и современный интерфейс** — Полноэкранная встроенная визуализация Tableau без отвлекающих элементов

## <a id="технологический-стек"></a> 🛠️ Технологический стек
- **Обработка данных:** Python (Pandas) + Jupyter Notebook
- **Визуализация:** Tableau Public
- **Фронтенд:** HTML5 + CSS3 + JavaScript (Tableau Embedding API)
- **Формат данных:** CSV / Parquet

## <a id="источник-данных"></a> 💾 Источник данных
Данные получены от **Домклик** и содержат среднемесячные цены за квадратный метр квартир в российских регионах с января 2022 по сентябрь 2025 года. Набор данных включает:
- **84 региона** Российской Федерации
- **2 категории недвижимости** (Первичный рынок: бизнес/эконом, Вторичный рынок)
- **45 месяцев** непрерывных данных

## <a id="как-использовать"></a> 🎮 Как использовать
1. **Откройте** файл `index.html` в вашем браузере
2. **Подождите** загрузки дашборда Tableau (оно того стоит!)
3. **Взаимодействуйте** с дашбордом:
   - Используйте фильтры для выбора конкретных регионов
   - Переключайтесь между типами рынка
   - Наводите курсор на точки данных для точных значений
4. **Анализируйте** тренды и поражайте коллег своей аналитической грамотностью

## <a id="обработка-данных"></a> 🧹 Обработка данных
Сырые данные были преобразованы с помощью Jupyter-ноутбука (`preprocessing_data.ipynb`) для обеспечения качества и согласованности:
- Разобрали колонку с датой на год и месяц
- Удалили обобщенные категории
- Убрали колонки с постоянными значениями (source, freq и др.)
- Исключили агрегированные данные по "России"
- Проверили на наличие отрицательных значений (не обнаружено!)
- Удалили дубликаты
- Экспортировали в форматы CSV и Parquet
